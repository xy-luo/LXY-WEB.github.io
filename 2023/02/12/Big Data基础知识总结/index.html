大数据的存储和计算 —— 计算

大数据的计算 Data Computation

海量数据处理时存在难题：

  1. 单机难以处理计算所有的大量数据；分而治之

  2. 分而治之的思想会涉及：计算机节点之间数据的交互（外存文件数据I/O访问制约系统性能）——计算靠近数据，减少数据传输。

  3. 代码靠近数据：

    • 本地化数据处理（locality），尽可能在同一个计算机节点处理本地磁盘所存储数据；
    • 尽量选择数据所在DataNode启动Map Task；
    • 减少数据通信，提高计算效率；

  4. 数据靠近代码：

    • 本地没有数据处理时，尽量从同一机架/最近其它节点传输数据进行处理（host选择算法）

Computation

  1. Batch processing
  2. Streaming Processing

MapReduce: a classic Batch Processing model
used to process and generate large data sets with a parallel, distributed algorithm on a cluster.

MapReduce：
step1: Map function
step2: Data sheffle
step3: Reduce

MR provides:

  • distributed & parallel execution model
  • Fault-tolerance
  • Status and monitoring tools
  • A clean abstraction for programmers

Map function

  • Records from the data source (lines out of files, rows of a database, etc) are fed into the map function as key*value pairs: e.g. (filename, line).
  • map() produces one or more intermediate values along with an output key from the input.

Reduce function

  • After the map phase is over, all the intermediate values for a given output key are combined together into a list.
  • reduce() combines those intermediate values into one or more final values for that same output key.

    1）先sampling，得到quantile/percentile(e.g. 10%分位数是v1，则v1是第一个threshold设置的值)；
    2）e.g. 10%分位数是v1，则v1是第一个threshold设置的值；低于v1值的打上tag1，shuffle时进入第一组；

其他大数据常见处理method

单机：

  1. Divide and conquer
  2. BitMap

shuffle:
The process of moving map outputs to the reducers is known as shuffling.
A different subset of the intermediate key space is assigned to each reduce node; these subsets, known as partitions, are the inputs to the reduce tasks.

集群中包括大量服务器，
分布式并行编程：

  1. 运行在由大量计算机(服务器)构成的集群上，实现并行处理能力；
  2. 通过向集群中增加新的计算节点，实现集群计算能力扩充。
    基础前提：待处理数据集可以分解成多个小的数据集，且可以并行进行处理小数据集（无相关性，彼此独立）

数据向计算靠拢：（将数据拉到应用程序所在的机器）

  • 需要大量的网络传输开销
    计算向数据靠拢：（将应用程序分发到数据所在机器）
  • 将计算节点和存储节点放在一起运行，减少节点间的数据移动开销

MapReduce采用Master/Slave架构
Master：运行JobTracker（负责任务调度，监控它们的执行；并重新调度已失败任务）
Slave：运行TaskTracker（负责执行由JobTracker指派的任务）

MapReduce作为一个批量计算框架，整个过程包括：数据输入、数据处理、数据输出

Map
每个Map任务通常运行在数据存储的节点上，这样不需要额外的数据传输开销；map结束后，会生成key-value pairs这样的中间结果。
中间结果被分发到多个reduce任务，具有相同key的key-value pairs会被送往同一个reduce任务；
reduce任务对中间结果汇总得到结果，输出到HDFS中。

不同的Map、Reduce任务间不会进行通信，所有的数据交换都是通过MapReduce框架自身实现；
Map Tasks的输出文件，Reduce Tasks的处理结果都是保存在分布式文件系统中的；
Map Tasks处理得到的中间结果保存在本地存储（如磁盘）中。

MapReduce Detail

  1. InputFormat模块做Map前的预处理：验证输入格式是否符合定义；将文件切分为逻辑上的多个InputSplit；
  2. InputSplit是对文件进行处理和运算的输入单位，只是一个逻辑概念，记录了要处理的数据的位置和长度，并未对文件做实际切割；
  3. 通过RecordReader(RR)根据InputSplit中对信息处理、加载数据并转换为适合Map任务读取对key-value pairs输入给Map Tasks；
  4. Map Tasks根据用户自定义的映射规则，输出一系列的<key, value>作为中间结果；
  5. Shuffle(为了让Reduce可以并行处理Map的结果)，对Map的输出进行分区(Partitioner)、排序(Sort)、合并(Combine)和归并(Merge)等操作，得到<key, value-list>形式的中间结果，交给Reduce进行处理；
  6. Reduce根据输入数据<key, value-list>执行用户定义的逻辑，输出结果给OutputFormat模块；
  7. OutputFormat模块会验证输出目录是否已经存在，以及输出结果类型是否符合配置文件中的配置类型；都满足，就将输出结果写入HDFS。

RecordReader记录阅读器，根据split的位置和长度，从HDFS各个块读取相关split，并转换成<k, v>的形式。

Shuffle

  1. Map端

    • Map的输出结果首先被写入缓存（缓存默认大小100MB）；
    • 缓存满了，启动溢写操作（把缓存中的数据写入磁盘文件 + 清空缓存）一般溢写启动threshold为80%，即80MB；
    • 启用溢写操作时，先把缓存中数据进行分区，然后对各分区里的数据sort&combine，再写入磁盘文件；
    • 每次溢写操作会生成一个新的磁盘文件，当Map Tasks全部结束时，这些溢写文件被merge成一个大的磁盘文件；
    • 然后通知相应reducer领取(fetch)其需要处理的数据。

  2. Reduce端

    • Reducer从Map端的不同Map机器领取(fetch)属于自己需要处理的那部分数据，然后对数据进行Merge后，进行Reduce处理。
    • Reducer将属于自己处理的分区的数据fetch回来，存放在自己所在机器的本地磁盘上；每个reducer会不断地通过RPC(Remote Procedure Call)向JobTracker询问map task是否完成；
    • 归并数据，map领回来的数据也是会先放在缓存，缓存占满，启动溢写操作，写入磁盘，此过程中具有相同key的pairs会被merge；
    • 数据输入reduce任务

MapReduce通过Partitioner接口对Map Tasks的输出结果key-value pairs进行分区；默认采用的分区方式是Hash函数对key进行Hash后，用Reducer的数量进行取模计算hash(key) mod R，把Map输出结果尽量均匀地分配给这R个Reducer并行处理。

分区后默认会对每个分区内的key-value pairs进行内存排序。

各分区内的combine操作是可选的，如果事先没有定义Combiner函数，则不进行combine操作；此处的combine操作可以减少需要溢写到磁盘和传输到reducer的数据量，类似reduce的操作。一般累加、最大值等场景可以在此使用combine操作。

所有溢写文件会生成一个大的溢写文件，其中所有key-value pairs也是经过分区和排队的；merge指对于具有相同key的pairs会被merge成一个新的key-value pairs，类似<k1, <v1, v2, v3, ...vn>>。
